# aigc-rewrite

> 学术文本 AIGC 降率改写 Skill,目标稳定将课题申报书/论文的 AIGC 检测率从 80%+ 降至 **25-35%** 区间。
> 适配 **维普 2026 多模型交叉验证**,兼容知网 AIGC、朱雀论文版。

## 项目简介

这是一个为 Claude Code 设计的本地 Skill,封装了完整的学术文本降 AIGC 率知识库与辅助工具:

- **改写指令**: 五大维度(句法/段落/密度/词汇/标点) + 学术专用第六维(批判性注入)
- **章节加权策略**: 摘要 ×1.8 / 引言 ×1.5 / 结论 ×1.5 等差异化处理
- **拼接预警规避**: 跨章节句长/人称/复杂度一致性检查
- **数据密集化**: 主动添加 β/p/R²/n 等精确统计反而降率
- **Python 辅助工具**: 指纹分析、长句拆分、AI 爱用词替换、章节切分等

**核心理念**: AI 洗稿 → 人工注血 → AI 润色 → 人工定稿,**必须人机配合**。维普 2026 可识别同一文本中的 AI 片段,所以【人工补充】部分必须真人手写。

## 通过 Claude Code 使用

### 方式一: 自然语言触发

直接告诉 Claude 你的需求,描述中含触发词(AIGC降重/降AI率/维普AIGC/洗稿/拼接预警/朱雀AIGC 等)即可自动加载本 Skill:

```
"我有一段论文摘要,维普查出来 78%,帮我降到 30% 以内"
"@thesis.md 这是文献综述,做批判性注入"
"整篇申报书做拼接预警检查"
```

### 方式二: Skill 命令

```
/aigc-rewrite
```

### 必要输入

Claude 会要求你提供以下信息(可在首条消息一并给出):

1. **待降率文本**(直接粘贴或文件路径)
2. **章节类型**: 摘要 / 引言 / 文献综述 / 方法 / 结果 / 结论 / 致谢 / 局限 (决定改写幅度)
3. 目标 AIGC 率(默认 25-35%)
4. 目标检测平台(默认维普;若多平台,优先按朱雀标准改)
5. 是否有可用的本地数据/访谈/统计结果(用于人工注血提示)

## 直接使用 Python 脚本

两个脚本无外部依赖,仅用 Python 标准库,Windows / macOS / Linux 通用。

### analyze.py — 指纹分析

```powershell
# Windows PowerShell
$env:PYTHONIOENCODING="utf-8"
python "C:\Users\hubo0\.claude\skills\aigc-rewrite\analyze.py" <文件>

# 章节加权评分(摘要 ×1.8)
python analyze.py --chapter 摘要 <文件>

# JSON 输出
python analyze.py --json <文件>

# 改写前后对比
python analyze.py --diff <旧文件> <新文件>
```

```bash
# POSIX
python ~/.claude/skills/aigc-rewrite/analyze.py <文件>
```

输出报告含: 平均句长、逗号:句号比、第一人称密度、AI 爱用词、TTR、引用密度、统计数据密度、机械列举警告、拼接预警、风险评分(0-100)。

### rewrite_helper.py — 改写辅助

```powershell
python rewrite_helper.py <子命令> <文件>
```

| 子命令 | 用途 |
|-------|-----|
| `locate` | 定位需改写位置(长句/AI 爱用词/并列结构) |
| `suggest` | 生成【人工补充】候选标记位置 |
| `split` | 长句拆分建议(基于标点) |
| `replace` | AI 爱用词替换建议 |
| `chapters` | 识别文档章节并按权重排序 |
| `cohesion` | **跨章节拼接预警检测**(句长/人称/复杂度) |
| `critique` | **文献综述批判性注入候选位** |

    │
    ▼  (AIGC 率 ~50%)

    │

    │
    ▼

## 核心原则

- **绝不替用户写【人工补充】内容**: 必须真人手写,保持人机分离。维普 2026 可识别同一文本中的 AI 片段。
- **章节加权差异化**: 摘要×1.8 / 引言×1.5 / 结论×1.5,改写幅度按权重分配。
- **关键章节单独达标**: 整体率不是唯一指标,关键章节风险等级必须为"低"。
- **拼接预警警觉**: 全文人称统一/句长一致(±5字)/用词复杂度均匀,低权重章节也要轻度改。
- **分阶段降率**: 先 AI 洗到 50%,人工注血到 35%,通读保留不完美到 30%。不要试图一次到位。
- **数据密集化反直觉**: 凡能写精确数据(β=0.47, p<0.01, R²=0.62)处必写,反而降率。
- **二次检测反升警告**: 不要用 ChatGPT/文心一言/Claude 二次改写已有 AI 内容,会强化 AI 特征。

## 实战案例

教育学硕士论文 12000 字:

| 阶段 | 整体率 | 摘要率 | 耗时 |
|-----|-------|--------|------|
| 初始(纯 AI 生成) | 58% | 76% | - |
| 第 1 轮(深度改摘要+引言) | 42% | 38% | 8 分钟 |
| 第 2 轮(标准改综述+设计) | 35% | 35% | 5 分钟 |
| 第 3 轮(人工微调) | **22%** | **28%** | 1 小时 |

**总耗时 1.5 小时**,相比纯人工 8-10 小时提升 5-6 倍。
